# CSI replay fixture (QA-1 / ESPA-173)

Deterministic, off-hardware replay of WiFi-CSI captures for regression-testing the
companion's (future) CSI / occupancy ingestion path in CI — the same idea as the
simulated trilateration harness next door, but for the CSI⊕BLE coexistence work
(ESPA-167 / ESPA-171). A captured corpus of binary CSI records is re-fed through a
sink seam at a configured rate and produces a repeatable digest, so coexistence
and occupancy logic can be regression-tested **without a board on the bench**.

## Status: SYNTHETIC interim corpus

> The 24 files in `corpus/` are **synthetic**, not measured. They are generated by
> a seeded model (`ecf1.py: gen_fixture`) that encodes a *plausible* BLE↔CSI knee
> (single-core esp32 starves BLE earlier than dual-core S3). They exist so the
> replay path is testable and CI-guarded today.
>
> The org's CSI⊕BLE spike resolved at a **modeled** knee — no physical RF bench
> exists, so the real captured corpus (ESPA-172) was never produced. A real corpus
> arrives later, from **prototype FW-1 milestone-1** (HVAC occupancy). When it
> lands, swap it into `corpus/`, re-cut `golden_digests.json`, and only then cut
> the *accuracy* golden in the C# AccuracyHarness.

What the golden here guards and does **not** guard:

| Golden | Lives in | Guards | Valid over synthetic? |
| --- | --- | --- | --- |
| `golden_digests.json` (this dir) | Python | ECF1 **replay determinism + byte stability** | ✅ yes |
| `csi-baseline-v1.json` (future) | C# AccuracyHarness | **occupancy / coexistence accuracy** | ❌ no — needs a real corpus |

## Division of labor (do not relitigate)

Same rule as `docs/accuracy.md`: **Python owns the bytes; C# owns the truth.**
The reference codec and the replay engine own the binary format and replay
determinism. The pass-criterion *metrics* and the *accuracy* golden stay in the C#
`AccuracyHarness` so there is no Python↔C# logic drift. The future C# `csi-replay`
verb consumes the same ECF1 container these tools emit.

## Files

| File | Role |
| --- | --- |
| `ecf1.py` | Reference ECF1 container codec + structural `validate()` + seeded synthetic `gen_fixture()`. Format oracle only. |
| `ecf1_adapter.py` | **§B → ECF1 adapter.** Converts the firmware's on-wire CSI frame (ESPA-172 "binary CSI frame layout v1", magic `0xC5`) + the §A per-bucket metrics into ECF1, off-hardware. `python3 ecf1_adapter.py` runs its self-test. |
| `run_ref.py` | Regenerates the 24-file synthetic corpus and prints an illustrative knee table. |
| `csi_replay.py` | **The replay engine.** `CsiSink` seam + deterministic virtual-time replay + CLI (`replay` / `manifest` / `golden`). |
| `test_csi_replay.py` | CI regression suite (stdlib `unittest`). |
| `golden_digests.json` | Locked replay digests (regression guard). |
| `corpus/*.ecf1` | 24 synthetic captures: 2 boards × 6 rates × 2 scenes. |

## Binary frame format

The full ECF1 v1 layout (64-byte `FileHeader` + tagged length-prefixed
CSI/BLE/STAT/MARKER records, binary — *not* ASCII, per esp-csi #249) is the
canonical [`csi-fixture-format` doc on ESPA-173](https://github.com/ESPresense/ESPresense/issues)
and is implemented in `ecf1.py`. BLE MACs are salted-hashed, so committed fixtures
are public-safe.

## Usage

```bash
# replay one capture at its native rate (human view)
python3 csi_replay.py replay corpus/esp32-s3_030hz_occupied-still.ecf1

# replay the whole corpus, force a rate, machine-readable
python3 csi_replay.py replay corpus --rate 50 --json

# corpus manifest (header summary + sha256 + digest per file)
python3 csi_replay.py manifest corpus

# re-cut the golden after a deliberate format/corpus change
python3 csi_replay.py golden

# run the regression suite
python3 -m unittest -v test_csi_replay
```

`--realtime` paces the replay in wall-clock time (for soak runs); it never changes
the digest — pacing controls *when* an event is delivered, not *what*.

## CI

`.github/workflows/csi-replay.yml` runs the suite on any change under this
directory. Python-only (no .NET) — this is the bytes/determinism layer; the C#
accuracy gate is `accuracy-harness.yml`.
